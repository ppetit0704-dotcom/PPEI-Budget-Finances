Version 2.1.0 Millésime 2025–2026 Auteur : Philippe PETIT Norme M57

📘 Introduction

PPEI-Budget&Finances est une application Windows standalone permettant aux élus, agents et responsables financiers de collectivités territoriales d'analyser leur situation budgétaire et comptable à partir des exports CSV issus de leur logiciel de gestion financière (norme M57).

Elle exploite deux fichiers sources :

  • La situation comptable (synthèse par chapitre avec historique N‑1 à N‑5)
  • Le Grand Livre détaillé par article (écritures individuelles)
💡 Aucune installation de Python requise. L'application est distribuée sous forme d'un installeur Windows (.exe) généré par PyInstaller + Inno Setup 6.

🚀 Démarrage rapide

En mode développement

pip install -r requirements.txt
streamlit run app.py

Via l'exécutable Windows

  1. Lancer PPEI-BudgetFinances.exe
  2. Le navigateur s'ouvre automatiquement sur http://localhost:8501 (délai ~8 s)
  3. Renseigner les chemins des fichiers CSV dans la sidebar, ou les importer via le bouton d'upload
  4. Sélectionner le budget souhaité dans le filtre principal

Chargement des données

Deux méthodes sont disponibles :

  • Chemin direct : saisir le chemin complet du fichier CSV dans le champ texte de la sidebar
  • Import : utiliser le bouton "Ou importer les fichiers" dans la sidebar pour téléverser directement depuis l'explorateur Windows
L'application détecte automatiquement l'encodage (UTF-8, UTF-8 BOM, Latin-1…) et le séparateur de champs (;, ,, \t, |). Aucune configuration manuelle n'est nécessaire.

📂 Fichiers CSV requis

FichierDescriptionEncodage détectéSéparateurVolume typique
Situation_new.CSV Situation comptable par chapitre — synthèse avec historique N‑1 à N‑5 UTF-8 BOM ; ~300 lignes
Grand_livre_détail_par_article.CSV Grand Livre — écritures individuelles (prévisions + liquidations) UTF-8 ; ~5 000 lignes
⚠️ Attention : Les deux fichiers doivent provenir du même exercice budgétaire. L'application filtre sur Libellé_budget pour isoler chaque budget (Communal, CCAS, Lotissement…).

🔤 Encodage & séparateur — Détection automatique

Le module core/data_loader.py effectue les détections suivantes à chaque chargement :

ÉtapeMéthodeRésultat typique
Encodage Librairie chardet sur les 8 premiers Ko Situation : utf-8-sig (BOM) — Grand Livre : utf-8
Séparateur Comptage des occurrences sur la 1re ligne (; , \t |) ; (point-virgule) dans les deux fichiers
Nombres FR Remplacement virgule → point, suppression espaces insécables Conversion en float64 pandas
Si votre logiciel exporte dans un encodage différent (Latin-1, Windows-1252…), l'application le détecte automatiquement et convertit à la volée. Aucune manipulation manuelle n'est nécessaire.

📊 Onglet — Vue d'ensemble

Présente une synthèse globale du budget sélectionné :

  • 4 métriques clés : Dépenses prévues, Dépenses réalisées (avec taux), Recettes prévues, Recettes réalisées
  • Par section F et I : jauges HTML de progression et graphique Prévu vs Réalisé groupé
  • Deux tableaux côte à côte — Récapitulatif par chapitre Dépenses (titre bleu) et Recettes (titre vert), avec Section, Prévu, Réalisé, Engagé et Taux

🏦 Onglet — Fonctionnement

Analyse détaillée de la section Fonctionnement (F) :

  • Sélecteur Dépenses / Recettes (radio button avec key unique)
  • 4 KPIs : Total prévu, Total réalisé, Total engagé, Taux global
  • Graphique horizontal Prévu vs Réalisé par chapitre
  • Graphique taux de réalisation coloré (vert > 60 %, orange < 60 %, rouge > 100 %)
  • Tableau détaillé avec colonne Disponible et Liquidé N‑1
  • Expander Détail par compte avec filtre chapitre

🔨 Onglet — Investissement

Identique à l'onglet Fonctionnement, appliqué à la section Investissement (I). Utilise le même module ui/tab_section.py avec section_code="I".

📈 Onglet — Évolution N‑5

  • Courbes F et I (Dépenses + Recettes) sur 6 exercices : colonnes Liquidé_N_5 à Liquidé_N_1 + Réalisé (N courant)
  • Affichage chronologique N‑5 → N (gauche → droite) — l'exercice courant est toujours à droite
  • Barres semi-transparentes + courbe avec valeurs annotées
  • Tableaux récapitulatifs par section en expander
  • Zoom par chapitre : sélection chapitre + sens → graphique barres N‑5 → N

📒 Onglet — Grand Livre

  • 4 filtres cumulables : type d'écriture (Prévision / Liquidation), Sens (D/R), Chapitre, recherche libre Tiers/Objet
  • 4 KPIs sur les liquidations filtrées : Nb écritures, Montant TTC, Montant HT, TVA récupérable
  • Top 10 tiers par montant TTC (liquidations)
  • Liquidations par mois — graphique recalculé dynamiquement selon le chapitre sélectionné, titre mis à jour automatiquement
  • Tooltip enrichi : montant TTC + nombre d'écritures par mois
  • Tableau complet des écritures (400 px, scroll) avec montants formatés

🗄️ Onglet — Tableau brut

  • Sélection source : Situation comptable ou Grand Livre
  • Filtres : Section, Sens, recherche textuelle globale (toutes colonnes)
  • Vue par chapitre avec expanders (Situation uniquement) — chaque expander affiche le résumé Prévu / Réalisé / Taux dans l'en-tête et le détail des comptes en deux colonnes D / R à l'intérieur, avec colonnes N‑1 et N‑2 si disponibles
  • Vue Grand Livre : tableau classique scrollable
  • Export en CSV (UTF-8 BOM, séparateur ;) ou Excel (.xlsx) via openpyxl

📐 Onglet — Ratios M57

Cet onglet est indépendant des filtres sidebar. Il dispose de ses propres paramètres :

  • Population : initialisée à 2 243 (INSEE RP2022, géographie au 01/01/2025)
  • Nombre de ménages : initialisé à 875 (INSEE RP2022)
  • Nombre d'emplois actifs : initialisé à 1 147 (INSEE RP2022)
  • Sélection multi-budgets : permet de choisir un ou plusieurs budgets pour les ratios

Organisation en 4 sous-tabs

🏗️

🧾

Chapitres utilisés

Cascade de calcul

MARGE_BRUTE    = (70 + 73 + 731 + 74 + 75 + 013) − (011 + 012 + 014 + 65)
ÉPARGNE_BRUTE  = MARGE_BRUTE − (66 + 67) + (76 + 77)
ÉPARGNE_NETTE  = ÉPARGNE_BRUTE − 16
DISPONIBILITÉ  = ÉPARGNE_NETTE + 002
Note 731 : Le chapitre 731 (Fiscalité locale) coexiste avec le chapitre 73 comme ligne distincte dans l'export. Les deux sont additionnés — pas de double-comptage, chacun représente une catégorie fiscale différente dans cet export.

📄 Export PDF

Accessible depuis le bouton "Générer le PDF complet" dans la sidebar. Généré avec ReportLab (sans kaleido ni dépendance externe).

Contenu du PDF

  1. Page de garde — titre, budget, date d'export, nombre de lignes
  2. Vue d'ensemble — tableau Prévu/Réalisé/Taux par section et sens
  3. Fonctionnement — détail par chapitre (Dépenses + Recettes)
  4. Investissement — détail par chapitre (Dépenses + Recettes)
  5. Évolution N‑5 — tableau historique par section
  6. Grand Livre — Top 30 tiers — montants TTC des liquidations

Un bandeau en-tête (budget + date) et pied de page (page / PPEI) s'affichent sur chaque page.

⚙️ Compilation PyInstaller

pyinstaller ppei_budget.spec --clean

Le fichier ppei_budget.spec configure :

  • Point d'entrée : run_app.py
  • Collecte complète Streamlit via collect_all("streamlit")
  • Inclusion des dossiers core/, ui/, assets/, app.py
  • Hidden imports : plotly, reportlab, chardet, psutil, openpyxl
  • Mode console=False (pas de fenêtre console Windows)
⚠️ La résolution des chemins en mode frozen utilise sys._MEIPASS. Toujours tester l'exécutable final après compilation — les assets peuvent nécessiter un ajustement de chemin.

📦 Inno Setup 6 — Installeur Windows

iscc ppei_budget_setup.iss

Paramètres clés du script ppei_budget_setup.iss :

  • Répertoire source : dist\PPEI-BudgetFinances\
  • Installation sans droits administrateur (PrivilegesRequired=lowest)
  • Raccourci Bureau optionnel
  • Lancement automatique après installation
  • Interface en français (French.isl)
  • Compression LZMA2 ultra
